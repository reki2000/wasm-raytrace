# コード解説

恐竜のレイトレーシング(SDF レイマーチングと、Quaternius メッシュの三角形レイトレース)
および群れ・歩行・スキンアニメーションの実装を理解するための解説です。UI・ビルド・
デプロイ手順は [README.md](README.md) を参照してください。
実装位置はすべて**関数名・定数名**で参照します(行番号は使いません)。

## モジュール構成

ソースは「レンダリングエンジン」「アニメーションエンジン」「恐竜モデル」「統合メイン」の4つに分割されています。

| ファイル | 役割 |
|---|---|
| `render.c` / `render.h` | **レンダリングエンジン** — SDF プリミティブストア(`PR[]`, `cone()` / `coneA()` / `boneCone()`)、距離評価(`mapL()` / `mapLE()`)、カリング(`buildList()`)、環境(`skyCol()` / `groundAlbedo()`)、ソフトシャドウ(`softshadow()`)、マテリアル状態(`M_*` / `setMaterial()`)、フレームパイプライン `renderFrame()` |
| `anim.c` / `anim.h` | **アニメーションエンジン** — アニメーションクロック(`animTick()`, `GT`)、ノースリップ歩行 `gaitFoot()`、群れ運動 `kin()` |
| `dino_model.c` / `dino_model.h` | **恐竜モデルとアニメーションパラメータ** — 3種のジオメトリビルダー(`theropod()` / `stego()` / `trice()`、脚 `tleg()` / `qleg()`)、群れ配置と太陽設定 `animate()`、種別テクスチャ `dinoAlbedo()`、群れの配置トグル `dinoSetActive()`(非配置時は `animate()` がプリム0件・境界球半径0で登録し、レンダラ側は何も知らないまま何も見つけない) |
| `mesh.c` / `mesh.h` | **メッシュレンダリングエンジン(第2経路)** — Quaternius glTF 恐竜用。アップロードバッファ(`meshPos()` ほか)、LBS スキニング `skin()`、BVH(`buildBVH()` を16フレーム毎、間は `refitBVH()` で境界のみ更新)、SoA 三角形ストア `packTris()`、4三角同時 SIMD の三角形レイトレ(`meshTraceScalar()` / `meshOccluded()`、Möller–Trumbore + スラブ)、環境のスカラー版(`skyCols()` / `groundCols()`)、フレーム `renderMesh()`。`meshTraceP()`(4レイパケット BVH トレース、原点も per-lane の `V3` なのでカメラレイにも床ミラーのような per-pixel 原点のレイにも使える)と `meshSurface()`(三角形 ID から法線・頂点色 albedo・マテリアルを引く)は `scene.c` から呼ばれる公開 API |
| `glb.js` | **GLB ローダ + glTF スケルタルアニメサンプラ**(Node/ブラウザ共用) — `parseGLB()` / `buildModel()`(プリミティブ平坦化・シーンフィット `computeFit()`)/ `sampleBones()`(キーフレーム補間 → ノード階層 → `逆バインド` → フィット合成 → 3x4 行列) |
| `scene.c` / `scene.h` | **統合シーンレンダラ(第3経路)** — SDF 群れとメッシュ列を**同一フレームに合成**する。`sceneRows()` は `render.c` の `renderRows()` を土台に、各ピクセルパケットで SDF スフィアマーチと `meshTraceP()` の両方を走らせ、より近い方を採用(遠い方は無視)。法線・albedo・マテリアル(`mode`/`refl`/`tran`/`ior`/`gloss`)はどちらの経路が勝っても同じ `v4` 変数に格納され、以降のライティング・フレネル反射・金属ミラーは**発生源を問わず同一コード**で処理される。太陽シャドウは SDF 側の `softshadow()`(発生源を問わない汎用関数)にメッシュ側の `meshOccluded()` 判定を掛け合わせて合成、床ミラーの再マーチも SDF・メッシュ両方を対象にする。アクリル屈折だけは表現が異なるため経路が分かれる(SDF は距離場内部を負距離でマーチして背面を検出、メッシュは面法線でスネル屈折して1回だけシーンをサンプル)。`scenePrep()` は `animTick()`+`animate()`+`meshPrep()` をまとめて呼ぶ。**このモジュール自体は「SDF」「メッシュ」という種別を一切判定しない** — どちらか一方だけを見せたい場合は `scene.c` に手を入れず、上流(`dinoSetActive()` / `meshSetCounts()`)でその群れ・列をシーンに配置するかどうかを切り替える(詳細は `template.html` の項を参照) |
| `main.c` | **統合メイン** — SDF 単独経路 `render`、メッシュ単独経路 `renderMesh`、統合シーン経路 `renderScene`(既定でテンプレートが呼ぶのはこれ)と `mesh*` アップロードエクスポート、`fb` / `mat`。`render()` は `animTick()` → `animate()` → `renderFrame()`、`renderMesh()` は `skin()` → `buildBVH()` → 走査、`renderScene()` は `scenePrep()` → `sceneRows()` |
| `vec.h` | 全モジュール共通の SIMD 数学層 — `v4`(f32x4)ラッパ、Relaxed-SIMD FMA(`vfma` / `vfnma`)、SoA ベクトル `V3` / `C3`、libm 代替スカラー数学(`fsin` / `fsqrt` など)、2D 値ノイズ `vnoise()` |

依存方向は `main.c` → 各モジュール、`scene.c` → `render.h` + `mesh.h` + `dino_model.h`。
レンダラはシーン固有の表面色だけをフック関数 `dinoAlbedo()`(`dino_model.h` で宣言)
経由で参照します。ホットな評価関数(`mapL()` / `mapLE()` / `buildList()` / `smin()` など)は
`render.h` に `static inline` で置かれ、どの翻訳単位でもインライン展開されます。
`skyCol()` / `groundAlbedo()` / `dinoMasks()` / `softshadow()` は(インライン展開は
狙わず)`render.h` で宣言・`render.c` で定義される通常の外部リンケージ関数で、
`scene.c` が SDF 側の環境・種別判定・シャドウをそのまま再利用するために公開されています。

SDF とメッシュの恐竜を1画面に共存させたい場合は `render()`/`renderMesh()` ではなく
`renderScene()`(MT ビルドでは `scenePrep()` + `sceneRowsSteal` の行ワークスチール)を
使ってください。`render()` / `renderMesh()` とその行並列版(`renderRowsSteal` /
`renderMeshRowsSteal`)は単独経路のみのテスト・後方互換用に残っています。

`template.html` は常に `renderScene()`(統合レンダラ)を使いますが、実際の見た目は
SDF 群れかメッシュ列のどちらか一方だけです。これは `scene.c` に「今どちらを見せる
か」を教える API(例: 種別トグル)を経由するのではなく、**見せたくない方をシーンに
一切配置しない**ことで実現しています: `dinoSetActive(0)` を呼ぶと `animate()` が
プリムを1つも登録せず(`DPR` 空区間・`DB` 半径0)、`meshSetCounts(0,0,jb)` を呼ぶと
メッシュ側の BVH が空になります。`scene.c` 側のコードは一切変更不要 — 「そこに何も
無い」ので単純に何も見つからないだけです。切り替えロジックは `template.html` の
メインループ(`sceneMode` の分岐)に閉じています。

## 設計理念

1. **決定論的・ステートレスなアニメーション** — 群れの位置・歩行位相・咀嚼などすべてが時刻 `t` の解析関数。フレーム間に状態を持たないので、どの解像度・フレームレートでも同じ絵になり、巻き戻し・ベンチも自由。
2. **SIMD は SoA(Structure of Arrays)で4ピクセル同時** — v128 の4レーン = 横並び4ピクセル。ベクトル `V3` は「x4本, y4本, z4本」の SoA 持ちで、分岐はレーンマスク(`sel()` / `vand()`)で表現する。`if (any(mask))` によりパケット全体が不要な処理(反射・影など)を丸ごとスキップできる。
3. **距離関数の Lipschitz 連続性を守る** — 全プリミティブは Lipschitz 定数 ≤ 1 を保証する形でしか変形しない(異方性は「太らせる」方向のみ)。これにより sphere tracing が絶対に面を突き抜けない。
4. **カリングの多段化で O(プリム数) を回避** — 個体ごとの境界球 → レイに近いプリムだけのリスト(`buildList()`)→ プリムごとの距離早期棄却、の3段でプリム84個を毎ピクセル評価しない。
5. **物理的に足が滑らない歩行** — 見た目の調整ではなく「接地中の足は地面フレームで厳密に静止する」というケイデンス拘束 K = π·ds/A を数式で守る(後述)。

## レイトレーシング・アニメーションの実装

### 1. SIMD 基盤とデータ構造(`vec.h`)

| 型 | 定義 | 役割 |
|---|---|---|
| `v4` (= `v128_t`) | `vec.h` | f32×4レーン = 4ピクセル分のスカラー値。`S(x)` で splat |
| `V3` | `vec.h` | `{v4 x,y,z}` の **SoA 3次元ベクトル**(4本のベクトルを同時に表す) |
| `C3` | `vec.h` | `{v4 r,g,b}` の RGB(4ピクセル分の色) |
| レーンマスク分岐 | `sel(a,b,m)` | `m ? a : b`(`wasm_v128_bitselect`)。以降すべての「if」はこれで表現 |

スカラー数学(libm 代替)も `vec.h`: `fsqrt()` / `ffabs()` は `__builtin_*`、`fsin()` は Bhaskara 型の放物線近似(SIMD 版は `vsin()`)。2D 値ノイズは `hash2()` + `vnoise()`(`hash = fract(sin(...)·43758)` の定番構成、2オクターブで使用)。

### 2. SDF プリミティブ: 異方性テーパーカプセル(`render.h` / `render.c`)

全ジオメトリ(84プリム)はこの1種類のプリミティブ + 目の球だけで構成されています。

- **データ構造**: `PR[MAXP][16]`。1プリム = float×16:

  | index | 内容 |
  |---|---|
  | 0-2 | 端点 a |
  | 3-5 | b−a(軸ベクトル) |
  | 6 | 1/&#124;b−a&#124;²(射影の除算回避) |
  | 7, 8 | 半径 r1, r2−r1(テーパー) |
  | 9-11 | カリング用中心 |
  | 12 | 境界半径(SK + 半長 + rmax) |
  | 13-15 | 異方性 Ax, Ay, Az |

- **登録**: `cone()` / `coneA()`。グローバル `OX` / `OZ` が「今ビルド中の個体」の原点オフセットで、各種ビルダーはローカル座標で書ける。
- **距離評価**: `mapL()`。`d = √(Ax·qx² + Ay·qy² + Az·qz²) − r(h)`。A≤1 でその軸方向に 1/√A 倍「太る」。**Lipschitz 定数は 1 以下のまま**なので sphere tracing 安全。制約: セグメント軸は A=1 の軸に沿わせること。ステゴの背板(z 薄・x 幅広、`stego()` 内の背板ループ)とトリケラのフリル(x 薄・y/z 幅広、`trice()` 内)はこれで各1プリム。
- **ブレンド**: `smin()`、滑らかさは定数 `SK = 0.062`(`render.h`。変更時は境界半径計算との整合に注意)。
- **目**: smin に混ぜず**ハード min の球**として別管理(`EYP` / `EYRAD` / `EB` と `eyePair()`、合成は `mapLE()`)。輪郭がにじまず、albedo/スペキュラの判定(`eyeDistAll() < 0.006`)にも再利用。
- **カリング**: `buildList()` がレイ区間 [tA,tB] にプリム境界球が接近するものだけを u8 リスト化。以降のマーチ・法線・AO はこのリストのみ評価。

### 3. 群れ挙動(`anim.c` + `dino_model.c` の `animate()`)

- **群れは実座標で前進**する。地面はスクロールせず、カメラの方が(メッシュ列と同じく)対象個体の実座標を追う(`template.html` が毎フレーム `dinoPosX/dinoPosZ` を読んで `meshSetFocus()` に渡す)。基準速度は `anim.c` の `VG = 0.85`。
- **各個体の前後ドリフト**は `kin()` が解析的に定義:
  `x(t) = x0 + D·sin(u + a·sin u)`, `u = ωt + φ`(スキュー正弦)。
  実効速度 `speed(t) = VG + dx/dt` が 0.72〜1.19 を往復し「ゆっくり遅れて、たまに駆け足で追いつく」動きになる。この振動は最大 2.0m に有界だが、実際のワールド座標(`animate()` の `OX`)にはさらに `VG·t` を足して累積前進させる — ノースリップ歩行(後述)が既に「積分済み移動距離 `dist = VG·t + drift` の分だけ足が地面フレームで静止する」よう作られているため、そのまま体を `dist` 分だけ実移動させれば足は滑らない。
- `Kin` 構造体(`anim.h`): `x`(振動のみの位置)/ `dist`(積分済み移動距離 = 歩行位相 & 実移動量の源)/ `speed` / `run`(0..1 の駆け足度。前傾・歩幅増などに使用)。
- **個体の登録**は `animate()`(`dino_model.c`): 種ごとにパラメータの違う `kin()` を呼び、`DPR[i]`(プリム範囲)、`DB[i]`(境界球)、`DXW` / `DZW`(実ワールド座標。テクスチャの原点にも、`dinoPosX()` / `dinoPosZ()` 経由でカメラ追従にも使う)を記録。太陽方向 `SUNX/Y/Z` もここで設定。

### 4. ノースリップ歩行(`anim.c` の `gaitFoot()` + 各種ビルダー)

`gaitFoot(ph, ds, A, *lift)`:
- 接地区間(位相 < デューティ `ds`)は足を移動距離に対して**線形に後送**、遊脚は smoothstep で前方復帰 + 正弦リフト。
- ケイデンスを **K = π·ds/A**(A = 半歩幅)に拘束すると、接地中の足は地面フレームで厳密に静止する(転がり接触と同じ原理)。
- ガイト位相は積分済み移動距離 `dist·K` から取る(`theropod()` の 5.585、`stego()` の 7.791、`trice()` の 8.469 がそれぞれの K)ので、**加減速中もスリップしない**。
- ⚠️ 歩幅 `A` や `ds` を変えたら、この係数も π·ds/A に合わせて更新すること。

### 5. 3種のジオメトリ(`dino_model.c`)

| 種 | 実装箇所 | 特徴 |
|---|---|---|
| 獣脚類 theropod | `theropod()`(脚は `tleg()`) | 二脚。**2ボーン IK**(`tleg()` 内で膝位置を余弦定理から解析解)。走行時前傾(`ln`)、顎のヒンジ開閉 `gape`、3本指の足 + 2本爪の手 |
| ステゴサウルス stego | `stego()` | アーチ体。**背板9枚**は `spineTop()`(体のカプセル鎖を正確にミラーした背面高さ関数)から基部を取り背骨に 3cm 埋め込み(尾側の板は尾スウェイ追従)。尾スパイク4本(thagomizer、`SPK[]` に記録)。四脚トロット(対角ペアで `qleg()`) |
| トリケラトプス trice | `trice()` | フリル = 異方性1プリム(中心高さは `FRLY`)、眉角2 + 鼻角(`TBN[]` に記録)、嘴、突進時に頭が下がる(`hd`)。四脚 |

共通: 四足の脚は `qleg()`(固定膝曲げ + 蹄トゥ2本)。全種、眼球2(高スペキュラ)+ 下顎(四足2種はゆっくり咀嚼 `chew`)。

### 6. SDF レンダリングパイプライン(`render.c` の `renderFrame()`)

処理順に:

| ステージ | 実装箇所 | 内容 |
|---|---|---|
| カメラ | `renderFrame()` 冒頭 | 注視点固定のオービットカメラ(az/el/dist)、焦点距離 FL=1.8 |
| レイ生成 | ピクセルループ先頭 | 横4ピクセルを1パケットに(`X4 = (0,1,2,3)`) |
| 個体境界球 | パケットごとの `DB[]` 交差判定 | 3個体の球とレイの交差。全ミスなら即スキップ |
| プライマリマーチ | `buildList()` → sphere tracing ループ | 絞ったリストに対し **64 iter** の sphere tracing。ステップ 0.92 倍の緩和、ヒット閾値は距離比例 |
| 法線・AO | ヒット後の `mapLE()` 4サンプル | **四面体法線**(4 サンプル)、AO は法線方向 1 tap |
| albedo | `groundAlbedo()` / `dinoAlbedo()` | 地面(チェッカー + grime ノイズ、ワールド座標に固定・スクロールなし)と恐竜を hit マスクで合成 |
| ライティング | 拡散 + アンビエント + スペキュラのブロック | 拡散(太陽1灯)+ 半球アンビエント + Blinn-Phong スペキュラ(目 1.4 / 恐竜は光沢スライダ、金属は 3.4 倍 / 床 0.05) |
| ソフトシャドウ | `softshadow()` | 個体ごとに境界球区間を作りマーチ(10 iter、`min(res, 9·d/s)` の古典式)して乗算合成(相互影が付く)。アクリル個体は透過率に応じて影を明るくする(`SH_LIFT`) |
| 体表の環境反射・マテリアル | `if (any(hit))` の反射ブロック | 反射方向の**解析スカイ + 地面チェッカー**をフレネル `F = (0.03 + 0.55·(1−cosθ)⁵)·2·反射率` で加算(再マーチなし)。骨色部位(角・嘴・スパイク)は光沢 1.5 倍。アクリルは屈折マーチ(後述)、金属は albedo で着色したミラーを反射率でブレンド(`mMet` ブロック) |
| 床反射(実マーチ) | `if (any(gm))` の床反射ブロック | 地面ヒット点から Y 反転レイで群れを**再マーチ(30 iter)**、影なし簡易シェーディング。ミス時は雲入り解析スカイ。鏡像にも金属/アクリルの見た目を近似反映(`rMet` / `rAcr` ブロック)。フレネル `kR = 0.14 + 0.60·(1−cosθ)⁵` でブレンド、影の中は減光 |
| 霧・空 | フォグブロックと `skyCol()` | 距離²ベースの霧、ミスピクセルは `skyCol()`(グラデ + 太陽グレア + 2オクターブ雲) |
| 出力 | 末尾のパックブロック | ガンマ 2.0(= sqrt)、RGBA8 パックしてフレームバッファに SIMD ストア |

### 7. テクスチャ / 種別の見た目(`render.c` の `dinoMasks()` + `dino_model.c` の `dinoAlbedo()`)

- `dinoMasks()`: マーチングに使ったプリミティブリストを個体ごとに実 SDF 評価し、最小距離の個体で種マスク m0/m1/m2 を作る(ヒットした部位のプリミティブは距離 ≈ 0 なので必ず正しい個体が選ばれる)。境界球中心への距離比較では、体の末端(ステゴの頭など)が隣の個体に誤帰属して色が化けるため不可。
- `dinoAlbedo()`: 種別パレット — 獣脚類 = 緑 + 背縞、ステゴ = 暖褐 + **背板の赤橙グラデ**(板ジオメトリを解析再評価する厳密判定 `PLT[]`。高さパラメータ h からグラデーション)、トリケラ = 灰緑 + フリル同心リング + 暖色リム。共通で 2 オクターブ値ノイズ変調 + 腹の明色化。縞・リング・ノイズは模様係数 `M_TEX`(スライダ)で 0(フラット色)〜1(フル)にブレンドされる。
- **部位マテリアルも座標ボックスでなくジオメトリ厳密判定**: 骨色部位(ステゴの尾スパイク・トリケラの角/嘴)はビルド時に `boneCone()` がセグメントを `SPK[]` / `TBN[]` に記録し、albedo 側で `segDist()` の距離 < 0.025 で塗る。フリルのリング/リムも実プリム(異方性円盤)への距離 < 0.02 で判定。軸平行の座標条件(例: 「lx が尾の範囲なら骨色」)だと部位からはみ出して隣の個体の色に見えるため不可。

### 8. 恐竜ごとの表面マテリアル(`render.c` + `renderFrame()` 内)

個体ごとに動的変更できる。状態はエクスポート `mat(i, mode, refl, tran, ior, tex, gloss)` 1本で受け(実体は `setMaterial()`)、個体別の静的配列(`M_MODE` / `M_REFL` / …)に保持。レーンへの展開は種マスク m0/m1/m2 越しの `dsel()` / `modeMask()` で行うので、1パケット内に異なるマテリアルの個体が混在してもレーンマスクだけで処理される。

| モード | パラメータ | 実装 |
|---|---|---|
| ノーマル | 反射率 / 光沢 | 模様係数 0 のフラット色。フレネル環境反射の倍率とスペキュラ強度をスライダで調整 |
| テクスチャ | 模様 / 反射率 / 光沢 | 従来の見た目(デフォルト)。模様スライダで縞・リング・ノイズの濃さを変更 |
| 金属反射 | 反射率 / 光沢 | albedo で着色した環境ミラーを `Fm = 反射率·(0.62+0.38·(1−cosθ)⁵)` でブレンドし、拡散を反射率に応じて減衰(`renderFrame()` の `mMet` ブロック)。スペキュラ 3.4 倍 |
| アクリル半透明 | 透過率 / 屈折率 / 反射率 | 入射面でスネル屈折 → **SDF 内部を負距離でマーチ(18 iter)**して裏面を検出 → 出射屈折(全反射時は直進継続)→ 出射方向の解析スカイ + 地面をサンプル(`renderFrame()` の `mAcr` ブロック)。厚みに応じて albedo 色を吸収近似でティント、透過率でブレンド。影も透過率に応じて明るくなる(`softshadow()` の `SH_LIFT`) |

屈折は環境(空 + 床)のみを対象にした1回バウンスの近似で、恐竜同士の透かし見はしない(コスト対効果の判断)。アクリル個体が視界にないパケットでは `if (any(mAcr))` で丸ごとスキップされるため、未使用時のコストはゼロに近い。

### 9. Quaternius glTF メッシュ経路(`mesh.c` + `glb.js`)

SDF 経路とは独立した**第2レンダラ**。[Quaternius Animated Dinosaur Pack](https://quaternius.com/packs/animateddinosaurs.html)
の三角形メッシュ(スキン付き・6体)を**横一列に並べて同時描画**する。中心にした個体に
アクション(6種)とマテリアル(SDF 版と同じノーマル/テクスチャ/金属反射/アクリル)を
適用する。既存のカメラ基底・チェッカー床・空・フレネル床反射・フォグ・ガンマ 2.0・
マテリアル意味論を**スカラーで再現**し、SDF シーンと切り替えても見た目が揃う。

計算の分担:

- **JS(`glb.js`)** — 起動後、初回のメッシュ選択時に6体の GLB を fetch/パースし、
  全プリミティブを1つの頂点/インデックス列に平坦化(色は `baseColorFactor`、UV・画像テクスチャは無い)。
  6体を**1つのストアに連結**(頂点/ジョイント/三角形のベースオフセットを足し込み、三角形ごとにモデル ID
  `TMDL` を付与、`fit` の平行移動に横一列のスロット X を加算)。毎フレーム各モデルで `sampleBones()` が
  選択アニメを時刻 `t` でサンプル(平行移動/回転のキーフレーム線形補間、回転は最短弧 nlerp)→ ノード階層で
  世界行列 → `逆バインド行列` → シーンフィット `computeFit()` を左から合成 → 3x4 行列を該当スライスへ書込。
  **フィットはメッシュ局所座標ではなくスキン後の参照姿勢 AABB から**算出する(スケルトンが大きなスケールを
  持つため。実装時の落とし穴だった)。
- **wasm(`mesh.c`)** — 連結済みデータを `skin()` が LBS で一括変形 → BVH は**16フレームに1回だけ**中点分割で
  再構築(`buildBVH()`)し、間のフレームはトポロジ据え置きでノード AABB のみ更新する **`refitBVH()`**(スキン変形は
  フレーム間で小さいので初期分割が有効なまま) → `packTris()` が **BVH 順の SoA 三角形ストア**を作る。
  一次可視性は**横4ピクセルのレイパケット**で処理する:`traceMeshP()` が1本のスタックを4レイで共有し、per-lane
  スラブ + per-lane best で最近接ヒットを求める(三角形をスプラットし1三角×4レイをレーン並列)。ヒット後の陰影は
  per-lane にスカラーで評価する(`shade()` → `shadeMeshHit()` / `shadeGround()`):太陽シャドウ・**床ミラー**(1バウンス)・
  **アクリル透過**(背後を再帰トレース)。単発の三角形交差(シャドウ/二次レイやリーフ)は **4三角形同時の SIMD
  Möller–Trumbore**(`traceMesh()` / `occluded()`)。法線は既定で**スキン後の面法線**(フラット)、マテリアルごとに
  グーロー補間を選択可。マテリアルは三角形の `TMDL` で個体別に引く。

エクスポートは `meshPos/meshJoint/meshWeight/meshIndex/meshColor/meshTriModel`(読込時に1回)、`meshBone`
(毎フレーム)、`meshSetCounts` / `meshSetFocus(x,z)`(カメラ中心)/ `meshMat(i,...)`(個体別マテリアル)/
`renderMesh(az,el,dist,w,h)`。上限は `MAXV 20000` / `MAXT 10000` / `MAXJ 200`(6×29ジョイント)。

低ポリ(6体計 約9500三角形)なので適応解像度で実用速度。交差は4三角同時 SIMD、BVH はフレーム間リフィット、
一次可視性はレイパケット化済み(元の未最適化比で約1.33倍)。残る余地は太陽シャドウ・二次・再帰レイ(床ミラー/
アクリル)のパケット化と、マルチスレッド(Web Worker + `SharedArrayBuffer`、要 cross-origin isolation)。ただし
リーフが既に SIMD 化済みのため、ここからのレイパケット化はブックキーピングが利得を相殺しやすく費用対効果は逓減する
(実測: シャドウのパケット化は約+1.4%止まりで、複雑さに見合わず見送り)。

## 調整ポイント(改修の入口)

- **群れ**: `animate()`(`dino_model.c`)の `kin(t, x0, D, ω, φ, a)` 呼び出し。D·ω·(1+a) がバースト速度を決める
- **歩幅/ケイデンス**: 各種の `A`(`qleg()` の amp / `tleg()` 内の 0.315)と `ds` を変えたらケイデンス係数を π·ds/A に合わせて更新(各ビルダー冒頭のコメント付きの箇所)
- **速度/品質**: プライマリ 64 / 反射 30 / 影 10 / 屈折 18 の各 iter 数(`renderFrame()` / `softshadow()` 内)。床反射の有無は `renderFrame()` の `if (any(gm))` ブロック
- **体型**: 各種ビルダー(`theropod()` / `stego()` / `trice()`)の `cone()` / `coneA()` 列
- **テクスチャ**: `dinoAlbedo()` のパレットと領域ルール(ローカル座標ベース)
- **ブレンド硬さ**: `SK`(`render.h`。変えたらバウンド計算との整合に注意)
